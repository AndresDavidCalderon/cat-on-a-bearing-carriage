## Text Elements
The driftiness of a shopping cart comes from
not moving the front of the car, but only the back.
How would we handle this mathematically, in code? ^trcqFQJw

The first step would be to save the position of the pin
on the world, so that we can set the car position
relative to the drift while we rotate. 
Rotation it seems can be kept normally while we drift, but we
also should move a little in the direction of the drift,
perpendicular to the side of the car. ^lRdUsxhy

Now: there is a tricky thing. We save the pin point, and then the position can
be calculated by moving back the offset. However we also want the car to move sideways, this
means that either the pinpoint needs to change or we need to change the 
way we calculate position.

One way would be to calculate the movement vector and apply it to the pinpoint slowly.
This movement should also invert with direction, which forces us to solve some deals:
First, when should we be able to drift?

- When we are going over a certain speed (No drifts from 0)
- Stopping rotation input or changing direction should stop a drift
- Drifting should only be started by pressing the button again.
- There is a rotation limit on how much you can drift, based on speed. ^NisZtvY8
